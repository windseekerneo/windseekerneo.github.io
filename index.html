github+hexo搭建的简易站点

By WindseekerNeo

Published 2017-02-05

从简书的这篇文章找到的相关技术，便于记录摘抄到这里。

准备

  • git
  • nodejs
  • hexo
  • 梯子

git和nodejs没有任何难度，即使是windows也已经有现成的安装包了。

hexo

通过命令行即可以安装
` npm install hexo-cli -g

同时，如果需要想时刻看到安装的进度情况，比如知道是否因为某墙的存在导致无法npm install，可以在执行前运行
` npm config set loglevel=http

编写

按照参考的文章，用hexo init，然后找个喜欢的主题，进行git clone。
$ hexo init username.github.io $ cd username.github.io
` $ git clone yourfavorate/hexo-theme
配置文件位于:username.github.io/_config.yml，主题的配置位于:username.github.io/themes/yourtheme/_config.yml.

测试

` hexo s

自动发布部署

安装自动部署工具：hexo-deployer-git
· $ npm install hexo-deployer-git –save
进行发布
· $ hexo clean && hexo g && hexo d
第一次发布会输入Github账户信息，以后发布直接运行这条即可。

关于梯子

梯子的必要性不言而喻，我的解决方案是使用ShadowSocks+Proxifier来进行。

proxifier

shadowsocks使用socket5，一般情况下只支持浏览器，为了能够全系统代理，比如cmd/npm/git，那么需要用proxifier来配合。
借用proxifier官网的说明
` Proxifier allows network applications that do not support working through proxy servers to operate through a SOCKS or HTTPS proxy and chains.
